설계: 라우트 구조 변경

Next.js Route Groups를 활용한 레이아웃 분리

현재 구조 (Before)
src/app/
  ├─ layout.tsx ← 루트 (테마)
  ├─ page.tsx ← 홈
  ├─ auth/
  │  ├─ layout.tsx
  │  ├─ login/page.tsx
  │  └─ signup/page.tsx
  ├─ dashboard/
  │  ├─ layout.tsx
  │  └─ edit/page.tsx
  └─ [username]/page.tsx
변경 후 (After)
src/app/
  ├─ layout.tsx ← 루트 (테마만)
  ├─ (public)/
  │  ├─ layout.tsx ← 헤더 + FAB
  │  ├─ page.tsx ← 홈
  │  └─ auth/
  │     ├─ layout.tsx ← Auth 전용
  │     ├─ login/page.tsx
  │     └─ signup/page.tsx
  ├─ (app)/
  │  ├─ layout.tsx ← 헤더 + 사이드바 + FAB
  │  └─ dashboard/
  │     ├─ layout.tsx ← 인증 체크
  │     └─ edit/page.tsx
  └─ [username]/page.tsx ← 레이아웃 없음
핵심 포인트
  • (public) — 홈, Auth 페이지들. 헤더 + FAB만
  • (app) — 로그인 후 페이지들. 헤더 + 아이콘 사이드바 + FAB
  • [username] — Route Group 밖. 글로벌 UI 없음
  • Route Group은 URL에 영향 없음 — /auth/login 그대로 유지
